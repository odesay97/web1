사용처에 맞는 태그 사용 --> 제목을 만들떄 /<h1>을 쓰는 것/과 /여러 태그를 써서 제목처럼 만드는 것/의 차이
----> 검색엔진이 검색할 당시 <h1>으로 된 키워드가 후자보다 우선순위가 높다
           --> <h1>은 제목을 위한 태그이므로 검색엔진이 이를 더 우선하여 찾아냈기 때문(후자는 그냥 키워드 하나가 포함된 느낌이라면 전자는 제목에 키워드가 있는 느낌이다.)
                     --> 이처럼 태그를 용도에 맞게 사용하는 것이 중요하다.

\\\\\\\\\\\\\\\\\\\\\\\  

<!doctype html>  --> 이 문서가 html로 이루어졌다고 명시하는 태그 
<html>
                         --> head, body등을 포함해서 모든 태그는 <html>태그로 감싸기로 약속
</html>

\\\\\\\\\\\\\\\\\\\\\\\\\



<head></head> -> html을 설명하는 태그들은 <head>태그로 묶기로 약속
			-> EX) meta, title 등등

<body></body> -> html의 본문에 들어갈 내용들에 해당하는 태그들은 <body>태그로 묶기로 약속

    
           --> 따라서 모든 태그들은 head 혹은 body 태그 안에 들어감(<html> 제외)

///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


<meta charset="utf-8">     --> 우리가 문서를 작성할 때 보통 utf-8 사용(한글 가능)
                                          --> 이때 이렇게 만들어진 html파일을 열 때 utf-8로 열지 않을 경우 한글 꺠짐
                                              --> 따라서 /이 html을 열 때 utf-8로 열어라/ 라고 지정해주는 태그(html이 utf-8언어 기반으로 열림)


//////////////////////////////////////////////////////////////////////////////////////////////////////////////
<a></a> -> anchor (닺->정보의 바다에 닺을 내림) -> Link를 걸어줌                     (                 중요                  )
      
<a href="www.naver.com"></a>   --> 어느 주소의 링크를 해당 글자에 걸어줄 것인지에 대한 정보 필요 ->  [ href ]
              -> 네이버 주소에 대한 링크가 걸림


      --> 이대로 할 경우 클릭시 해당 탭에서 열림
----> 새 탭에서 열리게 하기 위한 속성 [ target ]
<a href ="www.naver.com" target="_blank"></a>  ---> 새탭을 열어서 링크에 해당하는 주소(네이버) 열림


 ----> 링크에 마우스를 올려 놓을 떄 툴팁을 넣을 수도 있음 --->  [ title ]
 <a href="www.naver.com" title="이것은 네이버로 가기 위한 링크입니다."></a>   --> 해당 링크에 마우스 올려 놓으면 title에 해당하는 툴팁 나옴

<img> -> image ->이미지를 넣어줌
            --> 태그만으로는 불완전(어떤 이미지?? 등등의 정보 필요)  ----------->CSS
                  -> 태그에 필요한 정보 제공    [        attribute       ] (속성)   ----> CSS

<li></li>  -> list -> 목록으로 만듬 
                  --> 리스트를 여러개 하고 싶을 때 <li>태그의 부모태그인 <ul>태그로 <li>태그를 감쌈

<ul>                   결과 (앞에 점으로 나눠줌)  --> ul = unordered list
<li> 1 </li>         º 1
<li> 2 </li>         º 2
<li> 3 </li>         º 3
</ul>
                               

<ol>                         결과 (자동 넘버링)   --> ol = ordered list
<li> a </li>                1. a
<li> b </li>                2. b
<li> c </li>                3. c
</ol>
  
--> 서로다른 리스트로 인식  --> 따라서 기본적으로 li는 ul,ol등의 부모태그로 감싸자 !
